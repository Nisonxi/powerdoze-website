電腦風扇好吵？先別買散熱器，試試軟體

2026-06-12

夏天會讓每台電腦現出原形。整個冬天安安靜靜的機器，現在動不動就轉速拉滿準備起飛——視訊會議的時候、看網頁的時候，有時候根本不知道為什麼。

標準建議是花錢：更大的散熱器、更好的機殼風扇、換機殼。這些建議沒有錯——但它們治的是症狀。風扇的工作是搬熱、熱來自瓦數，而瓦數是個設定值。在花錢買「把熱更快搬走」的硬體之前，值得先試免費的選項：一開始就少產生一點熱。

風扇為什麼吵

CPU 和 GPU 把電變成運算，而幾乎全部都會變成熱。風扇控制器看溫度做事——瓦數越多、熱越多、轉速越高、越吵。這條鏈只有一個方向，意思是你可以從鏈的開頭下手，而不是結尾：

  • 硬體路線：同樣的瓦數、同樣的熱，搬得更有效率（買得好的話也更安靜）。
  • 軟體路線：更少瓦數、更少熱、風扇沒事做——代價是犧牲一些你可能根本沒在用的峰值效能。
風扇吵就像廚房的抽油煙機：火開得越大、油煙越多，它就得轉得越用力越吵。大家通常想的是換一台更強的抽油煙機（買更好的散熱），但更簡單的一招是先把爐火關小一點——油煙少了，抽風機自然不用拚命。風扇從來不是噪音的源頭，它只是在收拾瓦數產生的熱這個爛攤子。

讓電腦安靜的三個設定

  • 散熱策略改被動。Windows 的系統散熱策略（powercfg 的 SYSCOOLPOL，或進階電源選項裡的「系統散熱原則」）決定 Windows 先拉風扇（主動）還是先降 CPU 速度（被動）。被動模式是作業系統內建最直接的「安靜開關」——但只在 Windows 真的管散熱的機器上有效（主要是筆電和品牌機）；自組桌機的風扇歸主機板 BIOS 曲線管，這個設定作用有限，真正有感的是下面的 CPU/GPU 上限。
  • 限制 CPU 最大狀態。powercfg /setacvalueindex scheme_current sub_processor PROCTHROTTLEMAX 70 加 powercfg /setactive scheme_current，把 CPU 上限設在 70%——只要低於 100% 就會關掉 turbo boost，70% 還會再壓低一點持續時脈。不成比例的熱就藏在 turbo 裡：最後 20% 的效能要付出遠超過 20% 的瓦數。上網、看片、文書完全無感。
  • 給 GPU 設功耗上限。nvidia-smi -pl 150（系統管理員視窗）限制 NVIDIA 顯卡的功耗。重開機會重置，要靠開機任務。
turbo boost 就像開車最後那段硬要從時速 110 飆到 130：快不了多少，油耗卻暴增、引擎還吼得最大聲。CPU 不成比例的熱全藏在這裡——最後 20% 的效能要付出遠超過 20% 的電和噪音。把上限砍掉這一段（設成 70%），上網、看片、打字你完全感覺不到差別，風扇卻立刻閉嘴。

夏天還有加倍紅利：你沒燒掉的每一瓦，就是你的房間——和你的冷氣——不用處理的熱，省下的每一瓦也同時減輕冷氣的負擔。

一週內你就會撞到的問題

這些設定是靜態的，你的生活不是。讓開會時安靜得像圖書館的那組上限，會讓你下午四點轉影片時慢得想砸鍵盤、晚上九點打遊戲時直接抓狂。於是你來回切設定切個幾天，然後放棄，讓它繼續吵。問題從來不是設定本身——是切換。

該安靜的時候安靜，該快的時候快

這正是 PowerDoze 處理的問題形狀。把上面三個旋鈕（加上螢幕逾時）打包成電源模式——比如「安靜」模式：被動散熱+CPU/GPU 上限；「全速」模式：全部放開——然後讓自動化選對的那個：

  • 排程——深夜和會議密集的早上用安靜模式；下午的工作時段全速。
  • 離開模式——你離開座位它就安靜下來；回來瞬間恢復。
  • 全螢幕與 per-app 偵測——遊戲或算圖在前景就自動全速，不管排程說現在該是什麼時段，你完全不用碰。

結果是一台「你在乎安靜的時刻安靜、你在乎速度的時刻快」的機器——而且你再也不用打開控制台。（排程、兩個電源模式、全螢幕偵測和 GPU 功耗上限都免費；離開模式和 per-app 規則屬於 Pro。）

軟體幫不上忙的情況

誠實段：如果你的風扇連待機都在吼、溫度卻不高，那問題是機械性的——散熱鰭片積灰、風扇軸承快掛了、或 BIOS 風扇曲線設壞了。沒有任何設定能修物理問題；該清灰清灰、該換件換件。同樣地，一台薄筆電在持續 100% 負載下永遠需要風扇全開。軟體調校最有用的是最常見的那種情況：一台吵得跟它實際工作量不成比例的電腦——因為從來沒有人叫它冷靜下來。
