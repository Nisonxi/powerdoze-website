你的電腦到底吃幾瓦？三種量法，從免費到精準

Nisonxi

問一個人「你的電腦多耗電」，通常只會得到兩種答案：聳肩，或是電源供應器上印的那個數字。兩種都不對。750W 的電源供應器不會吃 750 瓦，就像時速錶標到 240 的車不會在市區開 240——那個標示是上限，不是讀數。

真實數字值得弄清楚，因為它幾乎從來不是大家猜的那樣——而且任何省電計畫的第一步，都是先知道自己現在站在哪。

沒人知道的那個數字

電腦的耗電會隨著它在做什麼，差到一個數量級：

  • 筆電上網、文書：大約 10–40 瓦。
  • 桌機閒置：插座端通常 50–100 瓦。
  • 遊戲桌機滿載：高階配置 300–600 瓦很正常。
問「電腦多耗電」就像問「一台車多耗油」——答案完全看你在做什麼。塞車怠速、市區慢開、跟高速公路猛踩油門，油耗可以差到十倍。所以「我的電腦吃 X 瓦」根本不是一句完整的話，它頂多是「此刻吃 X 瓦」。

同一台機器，十倍的範圍。所以「我的電腦吃 X 瓦」其實不是一句有意義的話——它是此刻吃 X 瓦。而且 Windows 不會告訴你：工作管理員有 CPU 百分比、記憶體、甚至 GPU 佇列的圖表，但整個介面裡找不到一個瓦字。

三種把數字弄到手的方法

  • 插座電錶——一個便當錢，買到真相。串接式功率計（經典的「電力監測插座」）或任何有電力監測的智慧插座，接在牆壁和電腦之間，流過多少就報多少——包括所有軟體看不到的部分：主機板、記憶體、硬碟、風扇、RGB 燈效，還有電源供應器自己的轉換損耗。量三個時刻——閒置、日常工作、滿載——你對這台機器的了解就超過絕大多數機主了。需要拿去做帳、報數字的，用這個方法。
  • 你桌下可能已經有的 UPS。多數家用 UPS 的面板或隨附軟體會顯示輸出瓦數。比較粗略，而且是接在電池端所有設備的總和——但如果 UPS 本來就在，它就是一顆免費電錶。
  • 軟體感測器——免費、即時，但只有半張圖。現代 CPU 會回報自己的「package power」，NVIDIA 顯卡會回報板卡功耗；HWiNFO、LibreHardwareMonitor 這類工具兩個都讀得到。但有兩個關鍵限制。在桌機上，CPU + GPU 不等於整台機器——漏掉的主機板、記憶體、硬碟、風扇和電源轉換損耗，讓插座端的真實數字往往比感測器加總高 30–60%。不過在用電池的筆電上，存在一個真正誠實的軟體數字：電池的放電速率就是整台機器的耗電，連螢幕都算進去——由實際付出這些電的硬體親自量出來。
桌機上靠軟體看 CPU + GPU 的瓦數，就像只算房子裡冷氣和冰箱的電費，卻忘了還有燈、熱水器、電視、Wi-Fi 分享器這一堆——主機板、記憶體、硬碟、風扇、加上電源轉換的損耗全沒算進去，難怪插座端的真實數字往往比軟體加總高出 30–60%。真正要精準，就在牆壁和電腦中間夾一個插座電錶（一個便當錢），那是房子的「總電表」，流過多少就誠實報多少。

把瓦數換算成錢

只需要一條公式：

每年度數（kWh）＝ 瓦數 × 每天小時數 × 365 ÷ 1000

一台閒置 120 瓦、從不睡眠的桌機，一年燒掉約 1,050 度電——以台電夏月常見的每度約 NT$4 計，大約 NT$4,200。這是它什麼都沒做的代價。

120 瓦聽起來不痛不癢，但它跟漏水的水龍頭一樣：一滴一滴你根本沒感覺，一年下來卻是滿滿一整缸。這台「什麼都沒做」的電腦，一年悄悄滴掉的差不多是 NT$4,200——足夠你吃好幾頓大餐了，而你連一秒鐘的用處都沒換到。省電的關鍵不是去算這個平均數，是找出那些「沒人在用、它卻全速空轉」的滴水時段，把水龍頭關緊。

但量測真正的價值不在算平均，而在找出浪費時段：機器用全速設定空轉、卻沒有任何人受益的那幾個小時。一台整夜吃 120 瓦的電腦，本來可以在限速的電源模式下吃 40 瓦——或在睡眠裡吃 2 瓦——這段差距就是所有省電空間的所在。（如果你的機器因為要跑 Plex 或掛載下載而真的不能睡，這段差距一樣關得起來——我們寫過一篇給常開機器的專文。）

把量測放在能動手的地方

知道數字本身一毛錢都省不到；省電來自狀態切換——需要時全速，不需要時限速。所以 PowerDoze 把量測直接做進負責切換的同一個 App 裡：

  • 儀表板上的即時瓦數。筆電用電池時，顯示的是電池放電速率量出的真實整機數字；桌機或插電時，顯示 CPU + GPU 感測器加總——而且 App 會標明你看到的是哪一種，不會假裝兩者是同一回事。
  • 逐模式的歷史紀錄。PowerDoze 按電源模式記錄功耗樣本，所以「安靜模式比全速模式實際省多少」是用你這台機器自己的數據回答的，不是型錄上的數字。
  • 用你自己的基準算省電。省電統計拿你配置過最耗電的那個模式當基準——而且只有在數據完全來自電池實測時才標「實測」，否則一律標「估算」。

接著由自動化——排程、離開偵測、全螢幕和逐 App 規則——負責切換，這是任何電錶都替你做不到的部分。

誠實角落：軟體數字適合拿來做什麼

軟體估算就是估算。桌機上的感測器加總天生少算插座端的數字；GPU 讀數涵蓋的是 NVIDIA 顯卡（AMD 和 Intel 的 GPU 沒有開放同樣的介面）；而且沒有任何 App——包括我們的——看得到你電源供應器的效率曲線。需要精準數字來報帳或請款，請買插座電錶；它的價錢比它一個月找出的浪費還低。

但對省電來說，重要的不是精準，是方向。「這個模式比那個模式少了 35 瓦」在軟體裡是成立的，即使兩個絕對值都偏低；而幾週下來的趨勢，正是告訴你一個改動有沒有用的東西。用電錶校準一次，用軟體每天盯著看。
