DP 螢幕一關，Windows 以為你拔了它——視窗亂跳與遠端斷線的元兇

2026-07-03

兩個常見抱怨，同一個元兇。第一個：多螢幕配置關掉其中一顆，上面的視窗全體大遷徙到還亮著的螢幕——圖示重排、版面全毀。第二個：想遠端連回一台螢幕全關的機器，串流工具給你一片黑、或一個可憐的 800×600 桌面。如果涉事的螢幕走 DisplayPort，這兩個都不是 bug——是 2006 年的協定設計決定。

HDMI 說「我關了」，DP 什麼都不說

兩種接口都有一條 hot-plug 偵測線（HPD）——告訴電腦「有螢幕接著」的那根訊號。差別在螢幕休眠或關機時的行為：

  • HDMI 螢幕通常保持 HPD 訊號。Windows 知道螢幕還在，只是暗了。桌面拓撲不動。
  • DisplayPort 螢幕常常在關機或深度休眠時把 HPD 整個放掉。對 Windows 來說，這跟你拔線一模一樣：螢幕不是關了，是不見了。

螢幕「不見了」，Windows 就做它對拔除螢幕該做的事：重排桌面。視窗搬到倖存的螢幕上、解析度可能改變；等螢幕醒來重新報到，整個重新枚舉一輪——就是螢幕回來時那段黑屏閃爍。你的 DP 螢幕會不會這樣，取決於型號和韌體：有的保持 HPD、很多不保持、有些在 OSD 裡給你選。

遠端版的同一個問題

把一台純 DP 桌機的螢幕全部關掉，這台機器就相信自己沒接任何顯示器。抓實體畫面來串流的工具——Parsec、Moonlight/Sunshine、部分 AnyDesk 模式——突然沒東西可抓，或掉到一塊迷你虛擬面板。（傳統 RDP 免疫：它自帶虛擬螢幕。）這就是遠端遊戲圈人手一顆 HDMI 假負載（dummy plug）的原因——一顆百元的電阻假裝自己是螢幕，讓系統永遠「接著一顆顯示器」。

解法工具箱

  • Windows 11 的視窗位置記憶。設定 → 系統 → 顯示器 → 多部顯示器 → 開啟「根據監視器連線記住視窗位置」。微軟就是為了 DP 消失術加的這個設定：螢幕回來時，視窗回到原位。它不阻止斷線，但幫你收拾善後。
  • 翻螢幕 OSD 找「DP 深度睡眠」。很多 Dell、LG、BenQ 型號有這個開關（有時叫 DP 省電）。把深度睡眠關掉，螢幕休眠時 HPD 保持活著——表現得跟 HDMI 螢幕一樣。代價是待機多吃一兩瓦。
  • 遠端族：假負載插一個閒置 HDMI 孔，或改用 RDP 而非抓畫面類工具。常態無頭跑的機器，假負載是最不會出意外的選項。

PowerDoze 在這件事裡的位置

PowerDoze 的電源模式可以設逾時關螢幕——走的是 Windows 標準的「關閉顯示器」，在行為正常的螢幕上完全可逆。但如果你的 DP 螢幕休眠就放掉 HPD，上面的視窗大遷徙在任何關螢幕時都會發生——不管是 Windows 自己的逾時、我們的、還是你按螢幕的電源鍵。沒有任何軟體能改變螢幕韌體怎麼處理 HPD 訊號。我們能做的是告訴你解法存在：開 Win11 那個設定、翻一下 OSD，多螢幕 DP 配置就能恢復正常。（睡眠設定的全貌與各種怪癖，見 Windows 睡眠設定完整指南。）

短版結論

DP 螢幕關閉 ≈ DP 螢幕被拔掉——這是協定，不是故障。視窗位置記憶治善後、OSD 深度睡眠開關治根、假負載救遠端。花二十分鐘設定一次，Windows 多螢幕最惱人的行為從此消失。
