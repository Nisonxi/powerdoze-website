主動 vs 被動散熱：Windows 最被誤解的電源設定

2026-07-10

Windows 進階電源選項的深處，躺著一個叫「系統散熱原則」的設定，兩個選項——主動、被動——零說明。我們最常聽到的猜測是「被動＝風扇不轉」。錯了，而且錯得情有可原：兩種模式風扇都會轉。這個設定決定的是更有意思的事：CPU 過熱時，Windows 先拉哪根桿子。

熱有兩個出口

晶片熱起來，系統只有兩招：把熱更快搬走（催風扇），或少產熱（讓晶片跑慢）。散熱原則就是這兩招的優先順序：

  • 主動：風扇優先。Windows 催散熱來撐住時脈——性能受保護，耳朵付帳。
  • 被動：降速優先。Windows 先降 CPU 速度少產熱，風扇留作最後手段——安靜受保護，峰值速度付帳。

真正的翻譯是：主動＝「先吵再慢」，被動＝「先慢再吵」。深夜掛下載的機器、開會時旁邊那台——被動是對的預設；趕渲染的機器——主動才對。

想像一個跑到發熱的人。主動散熱是「速度不變，把吹著他的電風扇開到最大」——很快跑完，但風扇吵翻天。被動散熱是「乾脆放慢一點、慢慢跑就不會那麼熱」——安靜又舒服，但會晚一點抵達。兩種電風扇都會轉；差別只在於你先伸手抓哪一個。趕死線衝刺？把風扇催到底。半夜散步？放慢就好。

在哪裡設

GUI：控制台 → 電源選項 → 變更計畫設定 → 進階 → 處理器電源管理 → 系統散熱原則（接電/電池可分開設）。指令：它是 powercfg 處理器子群組下的 SYSCOOLPOL。或者讓電源模式扛著它——下面細說。

細則：這個設定在哪種機器才真的有效

這是多數教學跳過的部分。散熱原則只在 Windows 自己管風扇的機器上才完整生效——實務上就是筆電和品牌整機，它們的韌體透過 ACPI 把熱控制權交給作業系統。典型的自組桌機，風扇聽主機板 BIOS 風扇曲線的，Windows 根本碰不到，所以「主動」那一半無從發力；剩下有效的是降速那一半——而那件事你用 CPU 上限控制得更直接。自組桌機要安靜，散熱原則是配角，CPU 上限和 GPU 功耗限制才是主角。

「會跟我的 BIOS 曲線或小飛機打架嗎？」

一位測試 PowerDoze 的讀者問了正是這題——他花了好幾個小時調 BIOS 風扇曲線和 MSI Afterburner profile，不想被任何東西蓋掉。答案讓人安心，值得講清楚：

  • BIOS 風扇曲線永遠不被碰。Windows 散熱原則（和 PowerDoze）做的是熱的「生產端」：CPU 時脈、上限、turbo。你的風扇繼續聽 BIOS 曲線——而因為到達感測器的熱變少了，同一條曲線自然轉得更慢。兩者是疊加，不是競爭。
  • Afterburner 可以共存，只有一個共用旋鈕。Afterburner 的風扇曲線、core/memory offset、電壓調校，PowerDoze 一律不碰。唯一交集是 GPU 功耗上限：兩邊都能設，後寫者贏。乾淨的配置：GPU 歸小飛機管的人，把 PowerDoze 模式裡的 GPU 功耗欄留空——留空＝完全不碰 GPU。
擔心散熱設定會蓋掉你辛苦調好的風扇曲線？其實剛好相反。你的 BIOS 風扇曲線是一台冷氣的溫控器：「熱到幾度，就吹多強。」散熱原則不去動那台溫控器——它只是把暖氣關小。房間進來的熱變少，同一台溫控器就會自己少開幾次風扇。兩者不打架；一個只是默默讓另一個更省力。

值得自動化的部分

散熱原則是個取捨，意味著正確的值跟著情境變：半夜被動、渲染主動、開會被動、遊戲主動。這正是沒有人會一天手動切五次的那種開關。在 PowerDoze 裡，散熱原則是電源模式打包的旋鈕之一——「安靜」模式帶著被動散熱＋CPU 上限，「全速」模式帶著主動＋全開——再由自動化（排程、全螢幕偵測、逐 App 規則）挑模式，散熱原則就跟著你的一天走，你不用記得它存在。

短版結論

主動＝先吵再慢。被動＝先慢再吵。兩種風扇都會轉。筆電和品牌機最有感；自組桌機真正做工的是 CPU 上限。還有——它不會吃掉你的 BIOS 曲線，也不會動你的小飛機 profile。
