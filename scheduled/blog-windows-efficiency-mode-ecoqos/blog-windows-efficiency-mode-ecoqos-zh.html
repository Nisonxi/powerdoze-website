工作管理員那片綠葉是什麼？Windows 效率模式（EcoQoS）解析

Nisonxi · 2026-07-17

在 Windows 11 打開工作管理員，有些程序旁邊掛著一片小綠葉。大多數人看過，幾乎沒人對它按過右鍵。那片葉子是效率模式——微軟給一個叫 EcoQoS 的排程等級取的名字——也是 Windows 裡最被低估的省電功能之一。

綠葉實際做了什麼

Windows 裡每條執行緒都帶著一個「服務品質」提示，告訴排程器這份工作有多要緊。EcoQoS 就是那個說「這工作不急——用最便宜的方式跑」的等級。具體上 Windows 會做三件事：

  • 優先排到效率核心。混合架構 CPU（Intel 12 代以後、AMD 近年設計）上，EcoQoS 的工作被導到 E-core（小核），讓 P-core（大核）閒著——而閒著的大核正是省電的來源。
  • 用更低的時脈跑。排程器不再為這些工作催 boost；同樣的指令，改在電壓-頻率曲線最便宜的那段執行。
  • 讓路給所有其他工作。程序同時降到低排程優先權，永遠不跟你正在做的事搶資源。
派一件不急的小差事，卻騎一台轟隆作響的重機去辦——又吵、又耗油，每個紅綠燈還硬要鑽到前面。效率模式就是跟這件差事說：「不趕時間，騎腳踏車就好、走小巷、讓所有趕路的人先過。」事一樣會辦完，只是不再為了辦它而燒油、又一路推開你真正在做的工作。

EcoQoS vs CPU 優先權：「品質」對「排隊順序」

傳統的程序優先權（閒置到高）只改變排隊順序——CPU 被搶的時候誰先上。一個閒置優先權的程序，在沒人搶的機器上照樣把 CPU 催到全速時脈、燒滿瓦數。EcoQoS 改變的是執行的品質：就算整台機器只有它，工作也是慢慢地、涼涼地、在小核上跑。優先權回答「誰先？」，EcoQoS 回答「花多少錢跑？」。對一台大部分時間閒置的電腦上的背景負載來說，真正能動到電錶的是 EcoQoS。

傳統優先權像銀行的叫號：它只決定誰先被叫到櫃檯。但如果整間銀行只有你一個客人，你照樣大步走到櫃檯、享受全套 VIP 服務——一點都沒省到。EcoQoS 不一樣：它跟那位客人說「走慢速通道、辦最陽春的服務、用走的別開車」。就算整間銀行任他用，也是用最省的方式打發。優先權＝誰先辦；EcoQoS＝辦得多奢侈。

它是給誰用的

所有「該活著、但沒資格快」的東西：

  • 夜裡磨著的下載和同步工具（雲端硬碟、torrent）
  • 掛在系統匣裡、帶著內嵌瀏覽器的聊天和啟動器 App
  • 索引器、備份代理、更新服務

模式很固定：你要它跑，但不要它花錢。一個每小時把 CPU 拉進 boost 幾百次的背景 App，掛上 EcoQoS 之後大多會安分下來——在一台原本閒置的機器上，殺掉那些 boost 尖刺就是實打實的省電，而且免費。

工作管理員的做法——和它的極限

工作管理員 → 對程序按右鍵 → 效率模式。可行，但有兩個坑。第一，它是「對這次執行」的：App 重開，葉子就掉了。第二，有些程序在那裡切不動，子程序還得一個個處理。當一次性實驗很完美；當每天的常規，就是苦工——又是這個部落格一直撞到的老問題：靜態設定 vs 變動的情境。

把它變成常設

在 PowerDoze 裡，處理程序管理（Pro 功能）把 EcoQoS、CPU 優先權和逐 App 的 GPU 偏好放在同一張清單——挑出那些該永遠便宜跑的程式，設定直接套用，不用每次重開機後去工作管理員尋寶。它跟工具箱的其他部分是互補的：電源模式按情境壓整台機器，EcoQoS 釘死那些在任何情境下都不安分的特定背景程式。（想知道它在 Process Lasso 那類工具——自動化規則深得多、複雜度也高得多——之間的位置，見我們的 Windows 電源工具誠實地圖。）

誠實角落

EcoQoS 是提示不是法律——Windows 執行得不錯，但省多少取決於那個程序本來在幹嘛。真正閒置的程序省不了什麼（它本來就便宜）；話多的 Electron App 省得有感；吃滿 CPU 的轉檔工作則會變慢而不是變省——別把你的渲染任務掛上葉子。另外，老的非混合架構 CPU 沒有小核可導，只剩時脈和優先權兩個效果。最佳目標是那些你會形容成「明明沒在幹嘛、卻一直很忙」的 App。
