筆電「睡著」還在偷跑電？元兇是 Modern Standby

2026-06-19

闔上螢幕，筆電塞進包包，出門。兩小時後拿出來：機身是溫的——有時燙得讓人緊張——電量少了一大截。沒有東西壞掉。你的筆電完全照著它的設計在運作。問題就出在這裡。

「睡眠」被換掉了

過去二十年，闔上螢幕等於 S3 睡眠：記憶體保持供電、其他全部斷電，整台機器只剩涓滴耗電，放幾天都還有電。後來 Windows 轉向 Modern Standby（S0 低功耗待機），現在大多數新筆電出廠就是它——而且韌體裡通常已經沒有 S3 這個選項了。

Modern Standby 的核心想法很像手機：「睡著」其實是淺淺地醒著。機器保持網路連線、同步郵件、下載更新、一碰就秒醒。運作正常時很美好；不正常時，某個驅動程式或背景工作把系統卡在較高功耗狀態——你的包包就成了保溫箱。健康的機器一夜只該掉幾個百分點；出問題的機器每小時可以燒掉 10% 以上，名為待機，實為醒著。

先診斷，再動手

開一個系統管理員終端機，直接問 Windows 它在幹嘛：

  • powercfg /a — 列出硬體支援哪些睡眠狀態。看到「待命 (S0 低電源閒置)」就是 Modern Standby；如果清單裡沒有 S3，代表韌體不提供傳統睡眠，任何軟體都加不回來。
  • powercfg /sleepstudy — 關鍵工具。產出一份 HTML 報告，列出每段待機：持續多久、吃掉多少電、哪些元件或程序讓系統閒不下來。答案通常就在「Top Offenders」清單裡。
  • powercfg /lastwake 和 powercfg /devicequery wake_armed — 上次是誰喚醒機器、以及哪些裝置有權喚醒它。

真正有用的處方

  • 解除流氓喚醒源的武裝。wake_armed 清單裡用不到的東西——通常是網路卡和雜七雜八的周邊——到裝置管理員（電源管理分頁，取消「允許這個裝置喚醒電腦」）或用 powercfg /devicedisablewake "裝置名稱" 關掉。
  • 待機時切斷網路。S0 背景耗電很大一部分是網路活動。部分機器的電源選項有「待機時的網路連線」這個設定；有的話，把電池模式設為停用，待機行為就會接近傳統睡眠許多。
  • 長時間離開改用休眠。休眠（S4）把記憶體寫進磁碟後完全斷電——零耗電，放多久、飛多遠都沒事。恢復多花幾秒，但對離開好幾個小時的人毫無影響。先確認休眠開著（powercfg /hibernate on），再把「用電池時闔上螢幕」的動作從睡眠改成休眠。
  • 登錄檔大絕招——附真實警語。網路上很多教學推薦 PlatformAoAcOverride = 0 整個關掉 Modern Standby。先想清楚代價：在韌體已不支援 S3 的機器上（最近幾年的筆電大多如此），你換不回傳統睡眠，只會得到一個未定義的狀態，而且 Windows 在部分新版本已經無視這個機碼。要試請充分測試、隨時準備改回來；對多數人來說，「闔蓋改休眠」就能達到同樣目的，沒有這些風險。

又是切換的問題

注意這些處方的共同形狀：休眠對包包是對的，對書桌是錯的。在桌上你要的是輕快的睡眠——進得快、醒得快；在路上你要的是完全斷電。Windows 只給你一組靜態的闔蓋和逾時設定，所以不管選哪邊，都有一半的時間是錯的。

這正是 PowerDoze 自動化的那一層。它不碰 S0 本身——它決定的是機器什麼時候、用什麼方式進入低功耗，而電真正省下來的地方就在這裡：

  • 在 App 裡設定闔蓋動作——睡眠、休眠、關機或不動作，接電和用電池分開設。「用電池時休眠、接上 dock 時睡眠」就是上面那帖包包處方的常駐版。
  • 用智慧任務排程休眠——設一個「01:00 自動休眠」，接住那些筆電忘了關就放著的夜晚——這是「早上電量莫名變少」最大宗的來源。
  • 逐模式的睡眠逾時——電池模式逾時抓緊、插電模式放寬，跟著電源模式自動切換，不用再進控制台。

誠實角落

任何第三方軟體——包括我們的——都無法改變機器進入 S0 之後 Modern Standby 本身的行為；那是韌體和 Windows 的地盤，要打那場仗請用上面的診斷步驟。軟體能負責的是決策層：確保機器每一次都在對的時刻進入對的狀態（睡眠、休眠、關機），而你不用動腦。實務上，大部分消失的電量一直都藏在這個決策層裡。想看 Windows 睡眠設定的全貌，可以讀我們的Windows 睡眠設定完整指南。
