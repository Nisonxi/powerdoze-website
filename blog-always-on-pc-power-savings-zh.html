24 小時開機的電腦怎麼省電——又不讓它睡著

Nisonxi · 2026-06-11

有些電腦是不准睡覺的：在維持分享率的做種機、全家都在看的 Plex / Jellyfin 伺服器、錄監控畫面的那台、幫朋友開的遊戲伺服器，或單純擺著讓你隨時能遠端連回家的跳板機。網路上每篇省電指南的結論都是「讓它睡眠」——對你來說，每一篇都沒用。

但重點是：「必須保持清醒」不等於「必須 24 小時保持全速待命」。這是兩個不同的設定，而 Windows 其實讓你把它們分開。

常開機的浪費長什麼樣

看看這台機器凌晨 4 點實際在做什麼：做種是涓涓細流的 I/O；媒體伺服器在沒人看片的時間基本上是閒置；監控錄影持續但很輕。大多數常開型工作絕大部分時間只需要機器幾個百分點的能力——但預設的「平衡」計畫讓 CPU 隨時準備加速、風扇照常運轉、獨顯醒著跑桌面時脈。

最常見的 homelab 模式還會讓這件事更糟：那台常開機就是你上一台遊戲桌機。為了峰值效能挑的硬體，現在做著一台 NUC 就能勝任的工作，全天候以 60–100 瓦空轉。深夜時段（比如 01:00–08:00）是最純粹的浪費：沒人看串流、沒人坐在桌前，而下載完全不在乎 CPU 能不能衝 5 GHz。

這就像為了偶爾載貨買了一台大引擎卡車，結果天天只拿來巷口買個醬油，引擎卻一路轟轟轟地催到最高轉速。深夜做種、半夜沒人看的媒體伺服器，需要的力氣大概是「腳踏車」等級，你的電腦卻整晚用「卡車全油門」在原地空轉——油（電）就這樣燒掉，貨（工作）一點都沒多送。

保持清醒的同時，這些旋鈕能降功耗

這些今天就能手動做。給喜歡 DIY 的人，配方如下：

  • 永不睡眠，但快點關螢幕。powercfg /change standby-timeout-ac 0 關掉睡眠；powercfg /change monitor-timeout-ac 5 讓螢幕 5 分鐘後關閉。無頭伺服器點亮螢幕（以及驅動它的 GPU）是純浪費。
  • 限制 CPU 最大狀態。powercfg /setacvalueindex scheme_current sub_processor PROCTHROTTLEMAX 50 再接 powercfg /setactive scheme_current，把處理器上限設在額定速度的 50%——turbo 完全關閉，時脈上限約在基礎時脈的一半。細流型工作完全無感，負載時的功耗和溫度直接降。
  • 散熱策略改被動。同一個 sub_processor 群組裡有散熱策略設定（SYSCOOLPOL）——被動模式會先降 CPU 速度再拉風扇轉速。注意：這只在 Windows 真的管風扇的機器上有效（主要是品牌機和筆電）；自組桌機的風扇歸主機板 BIOS 曲線管，真正讓它安靜的是上面的 CPU/GPU 上限。
  • 給 GPU 設功耗上限。NVIDIA 顯卡用 nvidia-smi -pl 120（系統管理員視窗）限制整卡功耗。注意重開機會重置，所以開機時要再跑一次。
「把 CPU 上限設成 50%」聽起來很可怕，其實就像在直線馬路上裝一個定速限制：你還是能正常開到目的地，只是車子不會再每遇到一小段空檔就猛踩油門催到 5 千轉。對只是涓涓細流的下載和待命工作來說，這個限速它根本感覺不到——但油耗（功耗）和引擎熱度（溫度）立刻降下來。

一個提醒：如果你的伺服器晚上有真工作——轉檔佇列、備份排程——上限設保守一點，或避開那些時段。

問題來了：這些設定全是靜態的

DIY 路線麻煩的地方在這裡。晚上 8 點你坐回這台機器想真的用它——結果它還卡在 50% 上限加被動散熱，因為那是你自己設的。於是你手動切回來。之後忘記再設回去。兩週後，這台機器又回到 24 小時全速，而你已經懶得再弄了。

誠實的 DIY 解法是兩個批次檔加上工作排程器：01:00 跑一個套用限制、08:00 跑一個解除，再加一個開機任務重新套 GPU 上限。這真的可行——如果你樂在維護，材料都給你了。它的極限也是老樣子：它不知道你此刻是不是正好在用機器、Windows 更新重置設定時它會無聲壞掉、而且你永遠不知道這一切到底省了多少。

用批次檔加排程「定時調快、定時調慢」，就像在客廳裝了一台便宜的定時插座：它確實會準時開關，但它是個瞎子——你晚上八點正坐在那台機器前面想好好用它，它照樣到點就把你降速，因為它根本看不到你人在不在。而且哪天 Windows 更新把設定洗掉，它也不會吭一聲，就默默壞給你看。

把夜班自動化

這個模式正是 PowerDoze 的核心場景。你定義電源模式——把 CPU 上限、散熱策略、螢幕逾時、NVIDIA 功耗上限打包成一組——再用排程規則按時段切換：白天均衡、01:00–08:00 切到限速省電模式。伺服器程式加進保持清醒白名單，從此任何規則、模式或未來的誤設定都不可能讓機器在做種到一半時睡著。GPU 上限自動重新套用，重開機、更新都不怕。

而這一切最該問的問題是「所以到底省了多少？」——所以有一頁分析儀表板（Pro 功能），長期追蹤估算瓦數、度數和電費：你這台機器的 before/after，不是假想數字。

排程規則、電源模式、白名單、監控全部免費、不用註冊帳號。一切都在本機執行，你機器上的任何資料都不會外傳。

誰不需要這篇

如果你的常開硬體是 Raspberry Pi、NUC 或功耗 10–20 瓦的專用 NAS，你已經用硬體解掉這個問題了——一篇講「怎麼削桌機待機功耗」的文章對你沒有用。這篇是寫給（非常常見的）另一種情況：因為軟體只能跑 Windows、或櫃子裡剛好有一台，於是拿舊桌機當伺服器的人。那台機器深夜有 20–40 瓦的肥肉可以削——而且削掉它，完全不需要讓機器睡著。
